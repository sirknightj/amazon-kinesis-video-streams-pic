◆ resetInstrumentedAllocatorsNoop()

STATUS resetInstrumentedAllocatorsNoop ( )

No-op equivalent of the resetInstrumentedAllocators.

NOTE: This is needed to allow the applications to use the macro which evaluates at compile time based on the INSTRUMENTED_ALLOCATORS compiler definition. The reason for the API is due to inability to get a no-op C macro compilable across different languages and compilers with l-values.

ex: CHK_STATUS(RESET_INSTRUMENTED_ALLOCATORS);
